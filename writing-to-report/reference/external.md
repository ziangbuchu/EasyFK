# 对外更新（客户 / 投资人 / 董事会）

这三种场合的共同点是：给**公司外的人**写。共同约束：

- **边界最严格** —— 不能暴露内部未决事项、人事动荡、战略调整中的细节
- **正式感要足** —— 拼写、格式、数字口径都要能经得起追问
- **坏消息要给，但要带方案** —— 纯报忧让对方焦虑，纯报喜下一次就被揭穿
- **数据和承诺要能对质** —— 外部会存档，后面的汇报会跟前面对

三种子场合各有侧重，下面分开讲。


## 客户更新

服务方 → 甲方。典型场景：月度进展邮件、里程碑交付汇报、事故通报。

### 特殊性

- 客户花了钱，你要让 TA 确信钱花得值
- 客户关心的是**交付**和**对 TA 业务的影响**，不是你内部怎么实现的
- 不能抱怨内部困难（人手、工具、系统），那是你的事
- 出问题要在客户自己发现之前先告知，并且带方案

### 基本结构

1. **TL;DR**：本月交付了什么 / 下月计划什么 / 关注点是什么
2. **交付进展**：按客户的业务目标组织，不是按你的模块清单
3. **关注点或风险**：如果有，事实 + 影响 + 方案
4. **下月计划**：让客户知道下周期能预期什么

### 模板

```
主题：XX 项目 3 月进展更新

本月交付：
- 订单管理模块上线，已处理 15w 订单，零故障
- API 响应 p99 从 800ms 降到 230ms

对业务指标的影响：
- 客服单量 -18%（订单相关问题减少）
- 结算延迟平均 -2.3 天

关注点：
- 跨境支付成功率 97.2%，低于 SLA 99%
- 原因：X 支付通道不稳定
- 应对：已接入 Y 作为 fallback，预计 4/15 恢复到 99%+
- 需要你方：无，定期关注即可

4 月计划：
- XX 模块二期（已排期）
- 报表系统上线（4/20）
- 跨境支付接入 Y 通道（4/15）

有任何问题欢迎直接找我或 account manager XXX。
```

### 禁区

- 不说"我们团队这个月人手紧张"
- 不说"内部在做架构调整"
- 不说"Stripe 那边的问题搞得我们很头疼"
- 只讲**客户关心的事**和**影响客户的数据**，内部困难是你的问题不是客户的问题


## 投资人更新

月度或季度投资人更新信，通常是邮件。

### 特殊性

- 投资人同时看几十个项目，你的信要能**快速扫**
- 投资人看的是**增长率 + 关键指标 + runway**，不是功能清单
- 既要诚实（长期信任），又要让投资人愿意继续支持你
- **Asks 是重点** —— 投资人能帮你引荐资源、背书、牵线，不用白不用

### 基本结构

1. **TL;DR** 一段：本月主题、核心数字、现金流状态
2. **KPIs**：MRR / ARR / 用户数 / 留存 / 增长率等硬数据，跟上期对比
3. **关键进展**：产品、团队、销售、合作
4. **挑战 + 应对**：诚实说出问题，带方案
5. **Asks**：具体到人或资源（"需要引荐熟悉 B2B SaaS 销售的 CRO 人选"）
6. **现金流**：剩余月数（runway）、本月烧钱、下轮时间表

### 模板

```
Subject: [Company] Investor Update - March 2026

TL;DR
- MRR $380k（+18% MoM），创历史新高
- 签下首个 $100k ARR 客户
- Runway 14 个月
- 需要你帮忙：B2B sales leader 人选引荐

KPIs
- MRR：$380k（+18% MoM，+290% YoY）
- 活跃客户：127（+23 net new）
- Logo churn：1.2%（跟上月持平）
- NRR：118%

关键进展
1. 产品：订阅套餐 v2 上线，首月 ARPU +32%
2. 团队：新增 3 人，HR + 2 eng，年底目标团队 22 人
3. 销售：pipeline $2.1M，覆盖未来 3 个月 target 的 3.5x
4. 合作：跟 XXX 签了 co-sell 协议（附件）

挑战
- Enterprise deal 成交周期比预期长（平均 94 天 vs 预期 60 天）
- 原因：客户采购流程复杂度被低估
- 应对：招聘专职 enterprise AE，正在面试 3 位候选人，计划 5 月到岗

Asks
1. 引荐熟悉 B2B SaaS 的 sales leader 人选（候选人最好带过 $10M+ ARR 团队）
2. 介绍 2-3 家 Series B 的 VP Sales，聊 benchmark
3. XXX 行业的客户 intro——我们的产品跟他们的场景很契合

现金流
- 本月 burn：$280k
- 账上：$4.2M
- Runway：14 个月
- 下轮计划：Q4 启动 Series A

谢谢一如既往的支持！如有任何问题欢迎随时聊。
```

### 禁区

- 别把失败包装成成功（"我们 pivot 了"真相是"上一条路走不通"）
- 别只讲产品不讲销售数据（投资人看的是生意，不是产品）
- Asks 别笼统（"希望得到你的建议"）——具体到人、资源、动作
- 数字口径跟上期要一致——投资人会做 Excel 横向对比，口径变了 TA 立刻发现

### 对"信任"的长期经营

投资人信任比客户信任更脆弱。一封信里的一个"美化"，下一轮融资的时候 TA 就会想起来。反过来，每个月老实汇报（包括坏消息 + 应对），下一轮融资的时候这些信就是你的信用账本。


## 董事会材料

董事会是公司最高决策层，汇报给董事会的文件叫 **pre-read**，会议开始前发出让董事成员提前阅读。

### 特殊性

- 董事会要**投票做决策**，不是听汇报
- 每个议题最后都要有"**需要董事会做的决定**"——没决策点的议题不值得占董事会时间
- 战略层面，不看执行细节
- 数字要最权威的版本（跟给投资人的一致，跟审计报告一致）
- 正式度最高，用词要经得起律师审查

### 基本结构

1. **Agenda**：本次会议议题清单
2. **每个议题一节**，每节结构固定：
   - **背景**：2-3 段，假设董事刚从别的会议切过来
   - **选项**：列出考虑过的方案 + 各自 pros / cons
   - **管理层推荐**：推荐方案 + 理由（3 条以内）
   - **需要董事会做的决定**：具体到一个 yes / no 或 A / B / C 选择
3. **附件**：财务报表、关键合同、法律意见

### 模板

```
# Board Meeting Pre-read — Q1 2026

## Agenda
1. Q1 financial review（信息性）
2. Series A 定价和条款审议（决策）
3. 高管招聘 CFO 审议（决策）
4. ESOP 扩容审议（决策）

---

## Item 2：Series A 条款审议

### 背景
- Q4 启动融资，本周收到 3 个 term sheet
- 估值区间 $60M - $85M post-money
- 本轮目标 $15M，稀释 15-20%

### 选项

| 选项 | Lead | 估值 | 条款要点 | Pros | Cons |
|---|---|---|---|---|---|
| A | Fund X | $85M | 1x non-participating | 估值最高，条款干净 | 要求 2 席董事席位 |
| B | Fund Y | $70M | 1x participating | Fund 品牌强 | 估值低，条款重 |
| C | Fund Z | $75M | 1x non-participating | 跟公司战略最契合 | Fund Z 第一个 B2B 项目 |
| D | 不做 | - | - | 保持现状 | 12 个月后 runway 吃紧 |

### 管理层推荐
推荐选项 A。理由：
1. 估值最高，后续轮次更有空间
2. 条款最干净，对员工 ESOP 价值保护最好
3. Fund X 在我们目标市场有 portfolio，有协同效应

### 需要董事会做的决定
批准选项 A、B、C 中的一个，或要求管理层继续谈判。
```

### 禁区

- 没有"决定点"的议题不要放进 pre-read（董事会时间金贵，纯信息性议题用邮件）
- 数字要跟季度报表对齐，不能现造
- 选项分析不能漏掉"不做"这个选项（很多场合"不做"是合理选项）
- 管理层推荐要给出来——董事会讨论的是"要不要同意推荐"，不是"请你们帮我选"


## 写完扫一眼（通用）

- 读者是谁？TA 的**信息需求**是什么？（客户要的不是技术细节，是业务影响；投资人要的不是功能列表，是数字；董事会要的不是进展，是决策点）
- 有没有暴露内部未决事项、人事动荡、战略模糊这类不该对外说的？
- 数据能不能经得起追问？口径跟之前汇报是否一致？
- 坏消息有没有**同时给方案**？
- 客户更新：有没有抱怨内部困难？抱怨要删
- 投资人更新：Asks 具体吗？数据口径跟上期一致吗？
- 董事会：每个议题有没有"需要你们做的决定"？没有的改成邮件
- 有没有把失败包装成成功？这个是长期信用的毒药
