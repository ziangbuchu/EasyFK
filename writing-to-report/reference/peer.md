# 跨部门同步 / 平级沟通

跨部门同步是向平级（或他部门）发起的汇报——跨团队邮件、协作协调、跨部门通知、横向项目更新。

跟向上汇报最大的不同是：**听者跟你没有权力关系**。TA 没义务关心你的事，不一定熟你的 context，ask 得是协商不是命令。

这种场合的特殊之处：

- 对方对你项目的 context **完全不熟** —— 不能假设 TA 知道
- 对方关心你这件事的**唯一动机**是：跟 TA 有关
- 对方有自己的优先级，你的事在 TA 那里排哪位，不是你能决定的
- Ask 是**协商**，不是命令


## 典型翻车

**用向上汇报的口吻** —— 显得施压或越权。平级看了会反感，回复变慢或者直接推诿。

**省略 context** —— 你觉得"这事大家都知道"，对方其实根本不了解你的项目。

**命令式 ask** —— "请尽快提供 X"、"需要你配合完成 Y"。平级没义务配合你，要问"能不能"、"什么条件下可以"。

**没讲"这事跟 TA 有什么关系"** —— 对方看了不知道为什么要花精力管你的事。英文管这个叫 WIIFM（what's in it for me），说的就是这个。

**没给备选方案** —— Ask 只有一个版本，对方说"做不到"就卡住了。给 2-3 个方案对方更容易接住。


## 基本结构

五段：

### 1. 主题行讲清事由 + 时间

主题不能是"协作需求"，要是"【协作】XX 功能接口对齐，请 4/20 前回复"。让对方收件箱一眼就知道这事什么性质、什么时间节点。没有时间节点的协作邮件最容易被拖——对方永远会先处理有 deadline 的。

### 2. 对对方的影响（第一段）

不是"我们在做 X 项目"，而是"**这件事对你们团队的影响是 Y**"。或者反过来："**我们需要你们做 Z，对你们的影响是 W**"。

让对方一眼看到跟 TA 有关——否则第一段过完，TA 就去处理下一封邮件了。

### 3. 快速 context（1-2 句）

即使对方可能知道，也给一个简短背景。别假设对方读过你之前的邮件、文档、或参加过某次会。

### 4. 具体 ask + 备选

- 什么时候、做什么、交付什么、给什么反馈——都要具体到日期和动作
- 给 2-3 个备选方案：理想 / 可以接受 / 真不行的 fallback
- 每个方案带上对双方的影响，让对方能评估

备选方案这一条很重要。平级协作最常卡在"对方说做不到，然后你俩都不知道下一步"。给备选就把"协商的球"踢回对方——TA 可以选 B 或 C，不必非得 A 或 zero。

### 5. 附件和接头人

讨论文档、相关设计、有问题找谁——都列清楚。让对方不用再回邮件问。


## 模板

```
主题：【协作】支付接口对齐，请 4/20 前回复

背景：我们 Q3 要上新的订阅套餐，需要改造支付流程。涉及你们 payment-service 的 2 个接口。

对你们的影响：
- Q3 需要你们新增 2 个接口（方案见附件 v0.3）
- 预计每天调用量 100w，对现有服务额外负载约 5%
- 方案对你们现有调用方无影响，只是新增

我们希望的节奏（按优先级排）：

方案 A（理想）：
- 4/20 前确认接口方案
- 5/1 前给到 staging 环境
- 5/15 前 production 上线
- 对你们的代价：约 3 人周的开发工作量

方案 B（可以接受）：
- 5/10 前给到 staging 环境
- 5/25 前 production 上线
- 我们会把产品上线推迟 2 周
- 对你们的代价：同 A，但时间更宽松

方案 C（如果都不行）：
- 我们自己在应用层模拟，性能会差
- 三个月后再回到你们的接口
- 对你们的代价：0，但我们背更多债

附件：
- 接口方案 v0.3（链接）
- 调用量预估（链接）
- 我方对接人：Alice（alice@），架构问题找 Bob（bob@）

有任何问题直接找我，或在文档里评论。谢谢！
```


## 特别提醒：不要用"请"和"需要"开头

"请尽快 X"、"需要你 Y" 这两种句式在上级对下级的场合很正常，但在平级里显得居高临下。

改成：

- "如果 X 在 Y 日前能完成，我们就可以…"
- "能不能请你帮忙 X？如果有困难，Y 方案也行"
- "希望听听你的想法——你觉得 X 还是 Y 更可行？"

口吻的一点点差别，对方回复速度能差出一倍。


## 写完扫一眼

- 主题行有没有讲清事由 + 时间节点？
- 第一段讲的是**对对方的影响**，还是在介绍你自己的项目？后者重写
- 有没有假设对方已经懂 context？
- Ask 是命令式还是协商式？"请尽快"、"需要你"改成"能否"、"希望"、"如果可以"
- 有没有给 2-3 个备选方案，每个带双方代价？
- 对方读完知不知道**为什么要关心这件事**？
- 有没有漏接头人 / 附件链接 / 讨论文档？
