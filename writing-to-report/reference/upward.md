# 向上汇报

向上汇报是所有汇报里最常见的一种——周报、月报、项目 review、OKR 对齐、一对一前的材料。听者是直接管你的人。

这种场合的特殊之处在于：

- 你和听者**共享一个 context**（公司战略、部门 OKR、项目背景），不用从零介绍
- 听者的视图按 **OKR / KPI** 组织，你的汇报要往 TA 的视图里靠
- 听者**同时管 N 个人 N 个项目**，你的汇报是 TA 整体视图里的一块，不是孤立的文章
- 听者要做的判断是**资源分配 + 绩效评估 + 继续或停止项目**——汇报要让这些判断有依据


## 典型翻车

**流水账** —— "本周做了 A / B / C"。没结论、没判断、没 ask，看完老板不知道项目处于什么状态。AI 写周报最常踩这个坑。

**只报喜不报忧** —— 关键数据藏在乐观包装里。老板后来从别的渠道知道真相，你的信用就开始透支。

**节奏跟老板要的不一致** —— 老板想看每周大事，你给每日细节；或者老板要 OKR 进度，你给功能清单。

**"推进中"综合征** —— 一切都在"推进中"、"有序进行"、"正在跟进"。这些词等于没说。

**跟 OKR / KPI 对不齐** —— 汇报里讲了 10 件事，老板看不出哪件对 TA 的目标有贡献。


## 基本结构

四段式，顺序不能变：

### 1. TL;DR（3 行以内）

- **状态**：一句话或一个颜色（绿 / 黄 / 红）
- **关键事件**：本周或本月最值得说的 2-3 件事
- **我希望你做什么**：批准、给资源、做决策；没有就写"无需动作，周期同步"

这三行是老板扫一眼就能决策的最少信息。后面全部是 TA 想深挖时的支撑材料。

### 2. OKR / KPI 进展

按老板视图组织。每个 O 一行，每个 KR 一行，给四个数：

- 当前数字
- 跟上期对比
- 按当前速度到期末能到多少
- 单项状态（绿 / 黄 / 红）

**不要按你的功能视图组织**。老板心里的坐标系是 OKR，你给 TA 功能清单 TA 还得翻译回 OKR，白白多一步。

### 3. 风险和卡点

放在显眼位置，**不要埋在最后**。每个风险四段，缺一不可：

- **事实**：什么情况，用数字
- **原因**：为什么发生
- **影响**：对 OKR 或目标的影响
- **建议**：你建议的应对方案 + 需要老板做什么

没有"建议"这一段就不叫汇报风险，叫甩锅。

### 4. 下期重点

老板看完这里要能知道你下阶段会花精力在哪里，好做人力和注意力分配。

下期重点要跟本周状态有逻辑关系——不是"我想做什么"，是"**基于当前状态，我下周应该做什么**"。


## 模板

```
TL;DR
状态：黄
本周关键：
- 用户增长达标，MAU 89w / 100w（89%），按当前速度月底可达 104w
- 支付集成延迟 3 天，因 Stripe 中国区稳定性问题
- 已完成切 Adyen 的 POC，迁移工期 5 天
我希望你：批准从 Stripe 切到 Adyen 的方案

OKR 进展
O1：用户增长
- KR1：MAU 100w → 本周 89w（89%），按当前速度月底可达 104w，状态 绿
- KR2：留存 40% → 本周 42%（105%），状态 绿

O2：支付系统稳定
- KR1：支付成功率 99.5% → 本周 98.7%（98.2%），状态 黄，见风险 #1

风险
1. Stripe 中国区稳定性
   事实：本周 3 次 downtime，累计 47 分钟
   原因：Stripe 中国区节点压力，已跟他们 support 确认短期无解
   影响：支付成功率掉 0.8%，用户投诉 +12
   建议：切到 Adyen。POC 已完成，迁移工期 5 天，停机窗口 30 分钟
   需要：你的批准

下周重点
1. 如果 Adyen 方案获批，周一启动迁移，周五切量
2. 留存上升做一次 cohort 分析，找出真实原因（当前只有感觉，没验证）
3. Q3 OKR review 准备，周五前出草稿
```


## 对"信任"的长期经营

周报是跟老板建立长期信任的最常用渠道。信任不是一次汇报建立的，是几十次汇报累计起来的。几个点值得注意：

- **说绿就是绿，说红就是红** —— 宁可早报红晚报绿，也不要"绿绿绿……突然红"。突然的红会让老板对你所有"绿"的汇报开始打折
- **Ask 要保守** —— 不要每周都要资源、每周都要决策。没必要的 ask 多了，真正要紧的 ask 会被 TA 忽略
- **坏消息自己先说** —— 别等老板从别的渠道知道。同样的事实，自己先说是"坦诚"，被别人说是"隐瞒"
- **承诺要兑现** —— 下周重点写的东西，下周要有交代。反复写"下周做 X"又不做，汇报就变成小说了


## 写完扫一眼

- 开头 3 行能不能让老板做决策？前 30 秒听完能不能回"批准 / 不批准"？
- 每个风险有没有事实 + 原因 + 影响 + 建议四段？
- OKR 进展是按老板视图组织的，还是按你的功能视图？
- 有没有"推进中"、"正在跟进"这种空话？全部改成带数字的事实
- 下周重点是"我想做什么"，还是"基于当前状态下周该做什么"？前者重写
- 有没有推测和事实混着写？标清楚
- 上周写的"下周重点"，这周有交代吗？没交代的要解释为什么
